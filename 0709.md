## 1. 목표 (Big picture)

RT-MRI 혀 영상에서 추출한 **midsagittal 움직임**을 실제 ArtiSynth 혀 3D surface mesh(`tongue_rest_m.obj`)에 옮겨 붙이는(**retargeting**) 파이프라인. 2-stage 구조:

- **Stage A — retarget**: 2D MRI 혀 마스크 → dorsal contour 추출 → image→model registration → RBF skinning 으로 3D rest mesh 변형. 폴더(비디오) 입력이면 프레임 전체를 retarget 하고 시간축 스무딩까지 적용.
- **Stage B — fem**: 11D 근육 활성값을 forward 시켜 혀 모델을 변형(ArtiSynth 대응 순수 파이썬).

retargeting = **모양 맞추기가 아니라 변위(displacement) 전달**이 핵심. `delta = target_contour − rest_contour` 를 모델 공간에서 계산해 Gaussian RBF 로 mesh 에 뿌린다. 그래서 model 과 MRI 의 절대 모양이 완전히 겹치지는 않는 게 정상.

`prev_work/` 는 같은 알고리즘의 **원본 스크립트 버전**(1~5번). 지금 작업은 이걸 패키지(`retarget/`, `artisynth/`, `modules/`)로 리팩토링 + 개선한 것.

움직이는 mask 실루엣(예전 vis_mask)은 제거함.

### 오늘 잡은 버그 3개
1. `dor[:,2]` → `dor[:,1]`: `model_dorsal_curve` 는 `[x, z]` **2열** 반환인데 3번째 열을 참조.
2. **프레임마다 모델 크기가 달라 보임**: midsag 패널이 축 범위 미고정 → 매 프레임 autoscale. 3D 도 box_aspect 없음. → 전 프레임 공통 범위로 고정(`set_xlim/set_ylim`, `set_box_aspect`).
3. **2번 패널이 아예 안 그려짐(빈 패널)**: 단위 불일치. `xl/zl` 은 `dm.verts`(**미터**)에서 계산했는데 midsag 데이터는 **mm**. → midsag 전용 mm 범위 `xmm/zmm = xl/zl × 1000` 를 만들어 2번 패널에만 적용. 3D(4번)는 verts 를 미터로 그리니 `xl/zl` 그대로.

### 주의: 단위 규약 (헷갈리기 쉬움)
- `dm.verts`(모델 정점) = **미터**. 4번 3D 패널·bounds(`xl,yl,zl`)는 미터.
- `model_dorsal_curve`, `midsag_dorsal_order`, `to_model()`, `mask2contour` 결과 = **mm**. 2번 패널은 mm(`xmm,zmm`).
- `rest_verts` 인자로 받은 것도 미터 → `Vr = rest_verts * 1000` 해서 mm 로 씀.

## 4. 핵심 알고리즘

**Retargeting (변위 전달)**
1. `mask2contour`: 2D 혀 마스크 → dorsal arc contour (image-mm). 버전(V1~V8)은 tip/root 검출 방식이 다름.
2. `register`: tip/dorsum/root 앵커로 image→model affine(lstsq) → `registration.csv`. 폴더 처리 시 **1회만**.
3. `delta = to_model(target_contour) − to_model(rest_contour)` (모델 mm).
4. spatial 스무딩(곡선 따라 `uniform_filter1d`, win=3) + **temporal 스무딩**(프레임축 `savgol_filter`, win=9, poly=2).
5. Gaussian RBF skinning: centers = model dorsal curve, values = delta, `epsilon=1/rbf_len(18)`, `degree=-1`, `smoothing=1e-3`. rest verts 에 x,z 변위 적용.

**파라미터**: NCTRL=13, RBF_LEN=18, SPATIAL_WIN=3, TEMPORAL_WIN=9.

**midsag 정점 고정**: 3D 모델의 "혀 중앙 라인"은 사용자가 수작업으로 고른 정점 `[418,416,409,408,414,406,422]`(tip→root)로 고정(`contour.midsag_indices`). 자동 추정이 tip 에서 엉키는 문제 때문.

**Contour 버전 (V1~V8)** — tip 오검출(혀끝을 감싸 바닥 고정부를 tip 으로 착각) 해결 위해 반복 개선. V8 = tip 은 normal corner + 좌측각 cap(V7), root 도 normal 이 우측 90° 직전을 root 로. 상세는 `versionmanagement.md`.

## 7. 이후 진행 (같은 날 후속 작업)

### 7.1 Contour 버전 V9~V11 추가 (`extract_contour.ipynb`)
tip/root 를 normal 회전으로 더 정교하게 자르는 버전들. 셀에 인라인, VERSIONS 그리드에도 포함.
- **V9** — V8 tip + root 를 **full boundary** 따라 posterior 벽으로 내려가 바닥-뒤 코너(col 역전, `back_tol`)까지 연장.
- **V10** — V9 + root 방향에서 normal 이 **반시계(CCW)로 급회전**(`root_ccw_deg`, plot 좌표계 각의 최단호 증가)하면 그 직전에서 끊음. 노치/코너 회피. (기본 20~30°, 사용자 선호 20°)
- **V11** — dorsum 기준 **양방향 대칭**: root=CCW 급회전 정지 + **tip 도 apex 를 감싸며 CW 급회전 직전 정지**(`tip_cw_deg`, 15°=apex만/25°↑=밑면까지). 각차는 `np.angle(exp(1j·Δ))` 최단호로 seam 안전.
- "root 근처 위로 솟은 부분" = 혀뿌리에서 혀(4)와 airway(5)가 만나는 **vallecula 노치 + 픽셀 계단** 아티팩트(별도 구조 아님).

### 7.2 DICOM → PNG 정리 (`datasets/prepare.py`)
`MRI_SSFP_10fps/Subject{n}/image_*.dcm` → **PNG 변환**(`png/image_{k}.png`) + 원본 dcm 이동(`dcm/`).
- **subject 단위 퍼센타일(0.5–99.5%) 정규화**(이상치 16-bit 스파이크 방지, 프레임 간 밝기 일관), RescaleSlope/Intercept·MONOCHROME1 처리, 재실행 안전.
- 실행됨: 5 subject·**392프레임** 변환. `--bits 16`(무손실)·`--root` 옵션.

### 7.3 혀 트래킹 — optical flow 라벨 전파 (`track_tongue.py`, `track_tongue.ipynb`)
혀가 **또렷한(clean) 기준 프레임**의 라벨을 grayscale MRI 의 Farneback optical flow 로 앞뒤 프레임에 전파.
- `propagate_labels(grays, order, refs)` — 각 프레임을 좌/우 최근접 기준에서 **양방향 전파 + 거리가중 블렌드**, 라벨별 soft map argmax 로 다중 라벨 지원(부위 body/floor/base 도 함께 전파).
- 검증: 단일 기준 Dice≈0.90, **기준 20프레임 간격+블렌드 → 5 subject 전부 Dice 0.93~0.95**.
- 한계: 붙어서 경계가 사라진(apposition) 프레임은 형상이 흘러들어와 채우지만 큰 변형엔 드리프트. 근본은 학습 모델(§7.6).
- `track_tongue.ipynb` — 오버레이 그리드 + 프레임별 Dice + 슬라이더 + 부위 전파 데모.

### 7.4 혀 부위(해부) 용어 + 분리 전략
한 라벨로 뭉친 혀를 나누려는 논의. 보라=**oral tongue body(구강부, 앞2/3, corpus+dorsum)**, 빨강=**아랫면+구강저(floor of mouth)**, 하늘=**혀뿌리/설근(pharyngeal part, 뒤1/3)**. 경계선: 몸통↔뿌리=**분계구(sulcus terminalis)**, 몸통↔구강저=genioglossus–geniohyoid 면(=예전 "검은 선", intensity 약함).
- 자동 intensity 분리(edge-path·dark-valley)는 5 subject 테스트에서 **근육-근육 경계라 불안정** → 반자동/학습이 현실적.
- 붙은 상태 = **apposition / articulatory closure(constriction)**; 영상분석 용어로 **touching objects / missing(weak) boundary**. 떨어진 상태 = airway(air–tissue) 경계 뚜렷.

### 7.6 Tongue Body **full-contour retargeting** (`retarget_body.py`, `retarget_body.ipynb`)
dorsal-arc 가 아니라 **닫힌 full contour** 로 retargeting → 기존처럼 **4패널** 시각화.
- **모델 full contour** = tongue mesh 를 **y=0 평면으로 slice**(삼각형-평면 교차 → chain)한 닫힌 midsag 외곽선. (midsag 정점 angle-order 는 내부점 때문에 엉켜서 폐기, slice 가 정답.)
- MRI body contour 를 registration affine 으로 model 공간 → **두 닫힌 contour tip(min x) 시작·같은 방향 정렬·N점 리샘플**로 점 대응 → `delta = body[k]−body[rest]` 스무딩 후 Gaussian-RBF skinning(centers=모델 full contour).
- 4패널: **MRI+body contour / model midsag full(회색 rest·파랑 변형모델·빨강 MRI) / 3D mesh(shaded) / 3D disp**. `body_retarget_4panel.gif`, max disp≈24mm.
- `retarget_body.ipynb` — 함수 인라인(수정용) + `show4(k)` 슬라이더 + GIF 셀.
- 개선 여지: (a) 대응을 tip/dorsum/root 랜드마크 구간별 정렬로 → 바닥 notch 불일치 완화, (b) 무게중심 정렬해 **translation 제거(모양만)** 옵션, (c) panel 3 을 사용자 머신 pyrender 로 교체.

### 7.7 학습(딥러닝) 방향 메모
- 자동 세그 모델(**nnU-Net** 권장)엔 라벨 필요하지만, 라벨 만드는 **SAM/SAM2/MedSAM 은 fine-tuning 없이 zero-shot**(클릭→마스크). SAM2 는 비디오 프롬프트 전파도 zero-shot(우리 flow 트래커 대체 가능).
- 양: binary 혀 ~subject당 15~30장(총 100~150), 3분할은 1.5~2배 + 라벨 규칙. **트래커 pre-label→어려운 프레임 교정**으로 비용 절감. leave-one-subject-out 검증(단 subject 5명은 적음).

## 9. 최종 목표 & 역문제(inverse) 접근 — 전체 그림

> §1~8 은 이 큰 목표의 **1단계(MRI→3D retargeting)** 와 그 도구들. 최종 목표는 아래.

**목표**: MRI 로부터 혀의 3D 움직임을 얻고, ArtiSynth FEM 으로 만든 합성 DB 와 매칭해
**어떤 근육이 어떻게 활성됐는지**를 라벨링 → 결국 **MRI ↔ 3D motion ↔ muscle activation** 라벨
데이터셋을 자동 구축한다. (관측 kinematics 로부터 근육값을 역추정)

**파이프라인 (4단계)**
1. **MRI → 3D retargeting** 을 거의 완벽하게 완성 → 관측된 혀 3D 움직임(mesh 시퀀스) 확보. *(현재 진행: §2~7)*
2. **FEM forward 대량 실행** → 근육 활성값(11D activation) 샘플링 → 각 결과 3D 변형을 저장.
   **(activation → 3D motion) DB** 생성.
3. retargeted 3D motion 의 **시작·끝(또는 전체 궤적)** 을 DB 와 비교 → **가장 유사한 motion** 검색 →
   그 엔트리의 activation 을 가져와 "근육이 어디서 어떻게 변했는지" 역추정.
4. 산출물 = **MRI–3D–MuscleActivation 라벨 데이터셋**.

**접근 성격**: analysis-by-synthesis / **retrieval 기반 inverse**. 정답(activation)을 아는 합성
motion 을 잔뜩 만들어 두고, 실제 motion 을 매칭해 activation 을 라벨링.

**핵심 이점 / 문제점 (설계 시 주의)**
- **정점 대응이 공짜**: retargeted 모델과 FEM 출력 둘 다 같은 `tongue_rest_m.obj`(433 verts) 토폴로지
  → vertex-wise 비교 바로 가능. 유사도 = 대응 정점 변위 차(평균 유클리드 / RMSE).
- **역문제 비유일성(muscle redundancy)**: 다른 activation 이 비슷한 motion 을 낼 수 있음 → 최근접 1개는
  불안정 → **k-NN 평균/가중** 또는 나중에 DB 로 **회귀모델(motion→activation) 학습**이 더 견고.
- **DB 커버리지/샘플링**: 11D 연속공간 전수 불가 → 격자보다 **Latin Hypercube / Sobol + 관심영역 집중**.
  **"시작·끝만" vs "전체 궤적"** 중 무엇으로 매칭할지가 정의를 크게 좌우.
- **정규화**: 매칭 전 rest 정렬(리지드/translation 제거 여부) 결정 필요 — §7.6 retargeting 의 translation
  이슈와 동일 선상.
- **DB 스키마(초안)**: `activation(11) | verts(433×3) 또는 시작·끝·요약 | 메타(파라미터, seed)`.
  motion 요약(예: 정점 변위 PCA, dorsum 곡선)으로 인덱싱하면 검색 빠름.

**다음 세팅 후보**: (a) **FEM DB 생성기**(activation 샘플링→`fem` forward→verts 저장), (b) **motion 유사도
매칭기**(retargeted↔DB, k-NN), (c) **retargeting 정확도 마무리**(선행 조건).
